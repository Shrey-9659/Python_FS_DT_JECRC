Studied about DRF : Django REST Framework
Django REST Framework (DRF) is a powerful toolkit for building Web APIs with the Python web framework Django.
Why use Django REST Framework?
Without DRF, you'd have to manually:

Convert Django model objects to JSON
Parse incoming JSON requests
Validate request data
Handle authentication and permissions
Build API endpoints and documentation

DRF provides these features out of the box.

How do we build an API using DRF
1. Install Django and DRF
code : pip install django djangorestframework
-----
2. Create a Django project and app:
code : django-admin startproject myproject
code : cd myproject
code : python manage.py startapp books
-----
3. Add DRF and your app to INSTALLED_APPS:
INSTALLED_APPS = [
    ...
    'rest_framework',
    'books',
]
-----
4. Create a Model
In books/models.py:

from django.db import models

class Book(models.Model):
    title = models.CharField(max_length=200)
    author = models.CharField(max_length=100)
    published_year = models.IntegerField()

    def __str__(self):
        return self.title
-----
5. Run migrations:

code : python manage.py makemigrations
code : python manage.py migrate
-----
6. Create a Serializer
A serializer converts model instances to JSON and validates incoming data.

Create books/serializers.py:

from rest_framework import serializers
from .models import Book

class BookSerializer(serializers.ModelSerializer):
    class Meta:
        model = Book
        fields = '__all__'

To get and post all data:
from rest_framework.response import Response --- Used to send the Response
from rest_framework.decorators import api_view 
from rest_framework import status --- To send status code
from .models import Student
from .serializers import StudentSerializer


@api_view(["GET", "POST"])
def all_students(request):
    if request.method == "GET":
        student_data = Student.objects.all()
        allstudents = StudentSerializer(student_data, many=True)
        return Response(allstudents.data, status=status.HTTP_200_OK)
    elif request.method == "POST":
        serializer = StudentSerializer(data = request.data)
        if serializer.is_valid():
            serializer.save()
            return Response(serializer.data, status=status.HTTP_201_CREATED)
        else:
            return Response(serializer.errors, status=status.HTTP_400_BAD_REQUEST)

Why use @api_view?
Without @api_view, a Django function receives a standard Django HttpRequest.

With @api_view:

Request becomes a DRF Request object
Automatic parsing of JSON data
Easy access to request.data
Supports DRF authentication
Supports permissions
Returns DRF Response objects
Provides proper API exception handling
